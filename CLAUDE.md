# CLAUDE.md — контекст проекта

Этот файл — память проекта. Claude Code читает его автоматически. Держись принципов отсюда.

## Что это

Интерактивный **иллюстративный** калькулятор риска ЗППП во времени. Показывает кумулятивную вероятность заразиться в зависимости от поведения, плюс визуализацию структуры партнёрств. Цель — объяснить логику риска, **не предсказать личный риск и не заменить врача**.

- Стек: **React + Vite + recharts**.
- **Весь код — в одном файле `src/App.jsx`.** Не дроби на модули без явной просьбы.
- UI на русском языке. Идентификаторы/код — на английском.

## Запуск, сборка, деплой

```
npm install      # один раз
npm run dev      # локальный предпросмотр
npm run build    # продакшен-сборка в dist (прогоняй перед коммитом, когда доступен Node)
```

- `vite.config.js`: `base: "./"` — относительные пути, чтобы работало на любом подпути GitHub Pages. Не менять без причины.
- Деплой автоматический: push в `main` → GitHub Actions (`.github/workflows/deploy.yml`) собирает и публикует на GitHub Pages.
- **Сборка перед коммитом — желательна, но не жёсткий блокер.** CI сам собирает на каждый push; job `deploy` зависит от `build` (`needs: build`), поэтому при провале сборки живой сайт остаётся на прошлой версии — сломать прод нельзя. Если в среде Claude недоступен Node (нельзя прогнать `npm run build`), Claude может коммитить и пушить после тщательной ручной вычитки кода, полагаясь на CI как страховку.
- Это обычный сайт (не артефакт Claude), поэтому `localStorage`/`sessionStorage` использовать **можно** (например, чтобы запоминать настройки).

## НЕЗЫБЛЕМЫЕ принципы честности (приоритет над всем)

1. **Предупреждение наверху не убирать и не смягчать.** Это любительский калькулятор, не медицинский инструмент; решения о тестировании/лечении — с врачом.
2. **Надёжны только данные по ВИЧ.** Вероятность передачи за акт (β) и эффективность презерватива для ВИЧ взяты из исследований. Для всех остальных инфекций надёжных per-act чисел в литературе нет — это правдоподобные оценки.
3. **Визуально различать данные и оценки:** ВИЧ — сплошная линия и маркер `●`; остальные — пунктир и `◌`. В таблице у каждой инфекции есть метка точности и источник (по наведению). Не выдавать оценки за точные числа.
4. Не добавлять числа/уверенность, которых нет в данных. Если что-то спекулятивно — помечать это в UI.

## Модель риска (формула — НЕ менять без явной просьбы)

Поведение задаётся **тремя типами партнёров** (`TYPES`/`BASE`): `steady` (постоянные, kind `ongoing`), `casual` (приходящие, `recurring`), `hookup` (хукапы, `oneoff`). У каждого типа свои поля: `count`, `condom%`, `tested%`, `perWeek`, `dur`, `poolMul`.

На один акт: `βeff = β · actMul · (1 − φ·e) · прививка`, где φ — доля актов с презервативом, e — эффективность, `actMul` — множитель вида секса (см. ниже), прививка — `veMul`. βeff клампится `Math.min(0.999, …)`.
Число актов с одним партнёром: `k = perWeek · (52/12) · длительность` (хукап = 1 акт; постоянный — длительность = текущий горизонт t, экспозиция копится).
Риск от одного заражённого партнёра: `transmit = 1 − (1 − βeff)^k`; шанс что партнёр заражён `pEff = p · poolMul · (1 − tested)`.
Вклад типа = `(1 − pEff·transmit)^count`. `casual`/`hookup` — годовая выживаемость, разворачивается `^(t/12)`; `steady` — длящаяся (k растёт с t). Общая выживаемость = произведение по всем типам → кумулятивный риск `1 − S`. Реализация — `survivalAt(s, t, cfg, veMul, actMul)`.

**Вид секса (`ACT_MUL`/`actMulOf`):** профиль «Виды секса» (state `acts`) — 3 практики × 2 роли: вагинальный/анальный/оральный × принимающий/вводящий(отдающий). Множитель к β относительно рец. вагинального (=1): рец.анал 17, ввод.анал 1.4, ввод.ваг 0.5, оральный 0.1/0.02 — **опора на ВИЧ (Patel 2014)**, для остальных инфекций грубое приближение (помечено в UI). `actMul` = среднее по выбранным практикам (равные веса). Рец. и ввод. вагинальный **взаимоисключаются** (анатомия). Дефолт `vagR=true` → actMul=1. Если ничего не выбрано → actMul=0, риск 0.

«Средний секс в неделю» — производная (`avgWeek`), показывается в статистике партнёрств, НЕ ввод.

### Параметры инфекций — в массиве `STIS` в `App.jsx`
Поля: `p` (распространённость), `beta` (передача за акт при заражённом партнёре), `e` (эффективность презерватива), `grounded` (true только у ВИЧ), `sev` (опасность 1–5), `treat`, `cons`, `src`, `acc`.
Инфекции: ВИЧ, ВПЧ, Гепатит B, Гепатит C, Сифилис, Гонорея, Хламидия, Трихомониаз.
Значения β/p/e для всех, кроме ВИЧ, — оценочные; правки величин ОК, но сохраняй пометки точности.

## Структура партнёрств (визуализация, НЕ часть формулы риска)

- Пресеты поведения (`PRESETS`): Целибат, Моногамия, Серийная моногамия, Monogamish, Открытые/свинг, Полиамория, ONS/хукапы, Core group. Каждый задаёт частичные настройки по типам (`mkCfg` накладывает их на `BASE`).
- Таймлайн (`buildPartnersTyped` + `packLanes` + `Timeline`): каждая связь — полоса от начала до конца; наложение по вертикали = одновременность (concurrency). Высота строк адаптивная. При большом числе связей отрисовка прорежается (≤240 полос/тип, спред на весь горизонт), но статистика (`built.total`) считается по всем аналитически.
- **Важно:** одновременность (concurrency) сейчас влияет только на эту визуализацию, а НЕ на формулу риска. Длительность связи — влияет на риск (через число актов с партнёром, см. модель выше). Эпидемиологически concurrency важна, но её строгий учёт требует сетевой модели. Не делать вид, что она уже в риске.

## Ключевые элементы UI

- Слева «студия»: пресеты → карточки трёх типов партнёров (`TypeCard`: count/презерватив/проверены + «детали» с perWeek/длительностью; при count 0 сворачивается в «+ добавить») → панель «Виды секса» (`SexActs`) + «Защита и иммунитет» (прививки ВПЧ/ГепВ). Справа — sticky график.
- Слайдеры графика: период активной половой жизни (горизонт, до 50 лет, НЕ возраст), масштаб шкалы вероятности. Средний суммарный секс/нед — производная (`avgWeek`), показывается в статистике партнёрств, НЕ слайдер.
- Всплывашки `i` (`.src .box`) — компактные, позиционируются JS-обработчиком в App (`mouseover`/`focusin`) с учётом краёв экрана.
- Таблица инфекций: чекбоксы (вкл/выкл линию), цвет-код опасности слева, риск к горизонту, передача за акт без→с презервативом, лечение, последствия, источник+точность по наведению.
- Панель «Разбор расчёта»: пошаговый вывод цифры для выбранной инфекции + наглядное сравнение «с презервативом / без» полосками. Текст про «расходование защиты» — условный: показывается только когда снижение реально падает со временем (для редких инфекций оно НЕ падает — это правильно).
- Палитра в объекте `C` (тёмный слейт). Форматтеры `pctVal`/`pctAct` — адаптивная точность для малых значений (не округлять 0,006% и 0,031% в одинаковое `<0,01%`).

## Известные ограничения / сознательные решения

- Три типа партнёров: число и длительность связи — полноценные драйверы риска. Фон среды (`poolMul`: 1.0/1.4/1.8) — оценка ассортативности, влияет на риск через `pEff`; помечено как оценка.
- Множитель вида секса опирается на ВИЧ, к остальным инфекциям применён как грубое приближение (один и тот же `ACT_MUL` ко всем) — помечено в UI и «Допущениях». βeff клампится, чтобы высокие β·actMul не уходили за 100%.
- Полоски сравнения масштабируются к большему значению в строке (показывают относительное снижение, не абсолют).
- recharts 2.x (npm предупреждает про 3.x — миграция отложена).
- `preview.html` (gitignored) — рабочий эскиз: открывается в браузере, рендерит 1:1 как сайт через CDN (esm.sh + Babel). Workflow: правки сначала в preview → одобрение → перенос в `App.jsx` (точечными Edit) + push.

## Источники (для справки)

- ВИЧ per-act: Patel 2014 (CDC) — рецепт. вагинальный ≈ 8 на 10 000, анальный значительно выше.
- Презерватив против ВИЧ ≈ 80%: Cochrane (Weller & Davis).
- Эффективность презерватива по другим ИППП: оценки CDC (гонорея >90%, хламидия 50–90%, сифилис 50–71%, герпес ~40%, ВПЧ — слабо).
- Числа передачи за акт для не-ВИЧ инфекций — оценочные, надёжных данных нет.
